# v1.3.0 — package split + Authy BC restore

**Date:** 2026-06-16
**Tag:** `v1.3.0` (minor) · **Plan:** [SDK package split & hardening](../plans/2026-06-16-sdk-package-split-and-hardening_plan.md)

A code review found the single `haakco/custd-sdk` Composer package autoloaded three
PSR-4 roots (pure PHP, Laravel, WordPress) and shipped all of them in the dist.
This release makes the root package pure PHP and turns the framework subtrees into
split-ready standalone packages. **Most consumers need no changes.**

## Changed

- **Root `haakco/custd-sdk` is pure PHP only** — autoloads `HaakCo\Custd\` →
  `sdk-php/src/`, requires just `php` + `ext-curl`, no `extra.laravel`.
- **Dist hygiene** — `.gitattributes` `export-ignore` strips framework subtrees,
  sibling-language SDKs, tests, and dev scripts. A generic consumer's install is
  now `sdk-php/` only (verified with `git archive | tar t`).
- **New split-ready packages** — `haakco/custd-laravel` (requires
  `laravel/framework`) and `haakco/custd-wordpress`, each resolving the SDK via a
  Composer path repo to `../sdk-php` while still in the monorepo.
- **Backward compatibility restored** — `v1.2.1` renamed `Custd\Authy\*` →
  `Custd\Awthy\*` and `redactAuthyAuditEvents()` → `redactAwthyAuditEvents()` in a
  patch, breaking downstream builds. The old class names and method are now
  **deprecated aliases** (lazy autoloader + delegating method) for one cycle.
- **Non-blocking retry** — new `retry.sleeper` option lets synchronous callers
  (e.g. WordPress request hooks) opt out of in-thread backoff that could stall
  page render.
- **CI** — added self-hosted `php-laravel` and `php-wordpress` jobs;
  `PackageBoundaryTest` / `PackagingTest` lock the split contract.

## Two notes for reviewers

- **`curl_close()` was intentionally not added.** It is **deprecated since PHP 8.5**
  and a no-op since 8.0 (handles are GC'd objects). PHPStan stayed green but the
  8.5 runtime emits `E_DEPRECATED`. The review item is resolved by documenting it,
  not by adding a deprecated call.
- **A dependency-declaration bug was caught and fixed during review.**
  `haakco/custd-laravel` first declared granular `illuminate/*`, but
  `SendCustdEvent` uses `Illuminate\Foundation\Bus\Dispatchable`, which ships
  **only in `laravel/framework`** (Laravel does not publish Foundation as a
  granular package; the Packagist `illuminate/foundation` is an unrelated
  abandoned v1.x). A `--no-dev` install would have fataled. Fixed by requiring
  `laravel/framework` and guarding it with `PackagingTest`.

## Deliberately left untouched

The review praised `FileQueueStore` (the on-disk producer queue) as careful
security work, and we preserved it byte-for-byte. For the record, here is exactly
what it does and why each part matters — so a future refactor doesn't strip a
safeguard by accident:

- **Atomic writes via temp file + `rename()`** (`save()`): events are written to a
  `tempnam()` file and then `rename()`d over the real path. `rename()` within one
  filesystem is atomic, so a concurrent reader or a crash mid-write never sees a
  truncated/half-written queue — it sees either the old file or the complete new
  one. Writing in place would risk a partially flushed JSON document.
- **Least-privilege permissions** — the queue directory is created `0700`
  (owner-only) and each temp file is `chmod 0600` before any data is written, so
  queued event payloads are never briefly world-readable.
- **Advisory locks** — `LOCK_EX` around writes and `LOCK_SH` around reads serialize
  concurrent processes sharing the same queue file, preventing interleaved writes
  and torn reads.
- **Symlink / path-swap guard (`assertSafeQueueFile()`)** — before reading,
  clearing, or overwriting, it rejects the path if it is a symlink or not a regular
  file, so the queue can't be pointed at `/etc/...` or another sensitive target via
  a planted symlink.
- **TOCTOU guard (`openExistingQueueFile()`)** — this is the subtle one. It
  `lstat()`s the path, opens it, then `fstat()`s the *open handle* and compares
  device + inode (`sameFile()`). If an attacker swaps the file between the check and
  the open (a time-of-check/time-of-use race), the dev/ino won't match and the read
  is aborted. Checking the path name alone would be insufficient; validating the
  already-open descriptor closes the window.

If you ever touch this class, keep the check-then-validate-the-open-handle ordering
and the `0600/0700` modes — they are the load-bearing parts.

## Upgrade

- **Pure-PHP consumers (e.g. Awthy):** nothing required; you stop pulling the
  framework/test bloat. The `Authy\*` names still work — migrate to `Awthy\*`
  before the next major.
- **Laravel/WordPress consumers:** continue consuming via the monorepo for now.
  Separate repos are a tracked follow-up
  ([repo split](../plans/future/2026-06-16-sdk-repo-split_plan.md)); no code
  changes will be needed when that lands.
