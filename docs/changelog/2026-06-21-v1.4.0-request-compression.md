# v1.4.0 — gzip request compression across all SDKs

**Date:** 2026-06-21
**Tag:** `v1.4.0` (minor)

All four SDKs now gzip-compress the request body on the batch ingest path, cutting
client egress and ingest inbound bandwidth ~5–8× for a typical batch (a ~30–45 KB
JSON batch goes out as ~5–9 KB). The custd ingest server already decodes
`Content-Encoding: gzip` (and zstd), so this lands without a server change.
**Backward compatible — most consumers need no changes.**

## Added

- **gzip request compression, default-on above a 1 KB threshold, configurable.**
  When enabled and the serialized batch body is at least the threshold, the SDK
  gzips it and sets `Content-Encoding: gzip`; below the threshold (or when
  disabled) it sends the raw JSON with no encoding header. `Content-Type` stays
  `application/json`. Compression covers only the batch send path.
  - **Go** (`sdk-go`) — stdlib `compress/gzip`; `ClientConfig.CompressionEnabled`
    (`*bool`, nil ⇒ on) + `CompressionThreshold` (default 1024). Both the `Doer`
    and `net/http` transports carry the compressed body.
  - **PHP** (`sdk-php`) — `gzencode()` (core `ext-zlib`, now declared in
    `require`); `compression: {enabled, threshold_bytes}` option.
  - **JS/TS** (`sdk-js`) — native `CompressionStream("gzip")` on the `fetch` batch
    path; `CompressionOptions {enabled, thresholdBytes}`. Falls back to
    uncompressed where `CompressionStream` is unavailable. The `sendBeacon` path is
    untouched (it cannot set `Content-Encoding`).
  - **Python** (`sdk-python`) — stdlib `gzip`; `compression` constructor option
    (`enabled`, `threshold_bytes`). Compression lives in the default transport.

## Notes

- **gzip-first, by design.** gzip is stdlib/native everywhere with zero new
  dependencies, and the server accepts it. **zstd** (better ratio where the runtime
  has it) is a documented follow-up, gated on no new dependency burden.
- **No new runtime dependencies** in any SDK. PHP adds an explicit `ext-zlib`
  requirement (already bundled in virtually all builds) for honesty, not new code.
